- On login, clinician fills biodata form
- then, clinician is redirected to fill verification request form
- Admin receives notification (signal) and reviews the clinician credentials
- Admin verifies the clinician
- Clinician receives verification notification (signal)
- After verification, on login, clinician is redirected to dashboard

Scenario 3: 

User Initially Registers as Patient but Later Becomes Clinician

This is the most important case.!!!!!!!!!!!!!!

Suppose:

User registers.
Completes Patient BioData.
Uses the system for months.
Later becomes a clinician or was assigned the wrong role.

Add a button:

Request Clinician Verification

Under:

Profile Settings

When clicked:

verification_requested = True
Show verification form.
Admin receives notification.
After approval:
role = "clinician"
is_verified_clinician = True

    patient
    ↓
request verification
    ↓
verification_requested = True
    ↓
admin reviews credentials
    ↓
admin approves
    ↓
role = "clinician"
is_verified_clinician = True

if request.method == "POST":
    user.verification_requested = True
    user.save()

    return redirect("clinicians:verification_review")

user.role = "clinician"
user.is_verified_clinician = True
user.verification_requested = False
user.save()

This prevents a patient from gaining clinician privileges before verification is completed.

A SIMPLE software (MVP) with the following features:
1. homepage
2. account registration with specific roles (admin can add users with roles)
3. users authentication with roles
4. patient's biodata & medical records
5. clinician's biodata
6. patient's symptom report
7. notification for clinician (for symptom report) & patient (for prescriptions)
8. messaging
9. prescriptions
