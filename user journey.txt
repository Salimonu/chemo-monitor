**User story

1. user visits the website HOMEPAGE
2. user SIGNS UP for a new account (default: patient) then redirected to dashboard
3. patient UPDATES biodata
4. patient UPLOADS medical records: diagnosis, treatment, lab reports
------------pay------------ 80/20
5. patient submits SYMPTOM REPORTS
6. clinician UPDATES biodata
7. clinician gets NOTIFICATION about the symptom reports
8. clinician VIEWS patient's symptom reports, biodata & medical records
9. clinician replies patient via MESSAGING (consultation)
10. clinician orders PRESCRIPTIONS
11. patient VIEWS the prescriptions


#####################
**Project Spec

A SIMPLE software (MVP) with the following features:
1. homepage
2. account registration with specific roles (admin can add users with roles)
3. users authentication with roles
4. patient's biodata & medical records
5. clinician's biodata
6. patient's symptom report
7. notification for clinician (for symptom report) & patient (for prescriptions)
8. messaging
9. prescriptions


########################
Real time features

- Clinician schedules APPOINTMENT with the client
- Send notifications to both patients and clinicians for appointment
- Clinician counsels the client via chat/audio CONSULTATION
